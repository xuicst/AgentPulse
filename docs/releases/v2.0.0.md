# AgentPulse v2.0.0

AgentPulse v2.0.0 is the first cross-platform release. It turns Claude Code and Codex Hook events into local desktop notifications on Windows, macOS, and graphical Linux desktops.

## Highlights

- **Windows x64**: native Windows Toast notifications with no .NET Runtime or Windows App SDK payload.
- **macOS Intel and Apple Silicon**: Notification Center through the built-in `osascript` command.
- **Linux x64 and ARM64**: native Helper using the current graphical session's D-Bus notification service; no `notify-send` requirement.
- **First-run setup**: AgentPulse can install the Claude Code and Codex Hooks from VS Code.
- **Fallback behavior**: if an operating-system notification fails, AgentPulse shows a VS Code notification instead.

## v1.0.0 compared with v2.0.0

| Area | v1.0.0 | v2.0.0 |
| --- | --- | --- |
| Platforms | One Windows-focused VSIX | Separate VSIX packages for Windows, macOS Intel, macOS Apple Silicon, Linux x64, and Linux ARM64 |
| Windows notification backend | .NET / Windows App SDK Toast Bridge | Small native Rust Helper |
| Windows VSIX size | 27.9 MB | About 202 KB (99.26% smaller) |
| macOS and Linux | Not supported | Implemented and built in native CI runners |
| Hook setup | Bundled Hook scripts | First-run enable commands in VS Code |
| Documentation | Chinese-only release note | English default documentation with Chinese, French, Russian, and Spanish translations |

## Install the correct package

Download the VSIX that matches your platform:

- `AgentPulse-win32-x64-2.0.0.vsix`
- `AgentPulse-darwin-x64-2.0.0.vsix`
- `AgentPulse-darwin-arm64-2.0.0.vsix`
- `AgentPulse-linux-x64-2.0.0.vsix`
- `AgentPulse-linux-arm64-2.0.0.vsix`

Install it from VS Code with **Extensions → ... → Install from VSIX...**, then reload VS Code. See the [main README](../../README.md) for first-use and privacy details.

## Validation status

All five platform packages are built by GitHub Actions on their matching native runners. Windows desktop notifications have been verified locally. macOS and Linux notification display still benefits from community testing on real graphical desktop sessions.
